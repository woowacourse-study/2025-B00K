# 아이템 6. 사용자 정의 오류보다는 표준 오류를 사용하라

- 예측하지 못한 오류: JSON 파일 형식에 문제 있다 → `JSONParsingException` throw 하기
- 이런 오류를 나타내는 적절한 오류가 없으므로 사용자 정의 오류를 사용하는 것이 좋다.
    - 직접 오류 정의하는 것보다 표준 라이브러리의 오류 사용하는 게 좋다.
        
        → 많은 개발자가 알고 있으므로, 다른 사람들이 API를 쉽게 배우고 이해할 수 있다.
        
- Exception
    - `IllegalArgumentException` / `IllegalStateException` : require, check 사용한 예외 처리
    - `IndexOutOfBoundsException` : 인덱스 파라미터의 값이 범위를 벗어났다는 예외 처리
    - `ConcurrentModificationException` : 동시 수정 금지했는데 발생했을 때 예외 처
        
        ```kotlin
        fun main() {
            val list = mutableListOf(1, 2, 3, 4, 5)
            
            for (item in list) {
                if (item % 2 == 0) {
                    list.remove(item)  // ConcurrentModificationException 발생
                }
            }
        }
        ```
        
    - `UnsupportedOperationException` : 사용하려던 메서드가 현재 객체에서 사용할 수 없다는 예외 처리
    - `NoSuchElementException` : 사용하려고 했던 요소가 존재하지 않을 때
        
        ```kotlin
        fun main() {
            val list = listOf<Int>()  // 빈 리스트
            
            val firstElement = list.first()  // NoSuchElementException 발생
        }
        ```
