## 사용자 정의 오류보다 표준 오류를 사용하라

직접 오류를 정의하는 것 보다 최대한 표준 라이브러리의 오류를 사용하는 것이 좋다.

잘 만들어진 규약을 가진 널리 알려진 요소를 사용하는 것은 다른 사람들이 API를 더 쉽게 배우고 이해할 수 있다.
```
IllegalArgumentException과 IllegalStateException : require 또는 check 를 사용해 예외를 던질 수 있는 예외
IndexOutOfBoundsException : 인덱스 파라미터 값이 범위를 벗어났음을 나타내는 예외
ConcurrentModificationException : 동시 수정(concurrent modification)을 금지했을 때 발생하는 예외
UnsupportOperationException : 사용자가 사용하려고 했던 메서드가 현재 객체에선서 사용할 수 없다는 걸 나타내는 예외
NoSuchElementException : 사용자가 사용하려고 했던 요소가 존재하지 않았을 때 발생하는 예외
```

이처럼 표준 라이브러리의 오류는 많은 개발자가 알고 있으므로 이를 재사용하는 것이 좋다.
