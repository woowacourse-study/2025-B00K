# 아이템 9. use를 사용하려 리소스를 닫아라

- 자동으로 닫히지 않는 리소스가 더 이상 필요하지 않다면 명시적으로 close 를 시켜야 함.
    - InputStream, OutputStream
    - jaja.sql.Connection
    - java.io.Reader(FileReader, BufferedReader, CSSParser)
    - java.new.Socket, java.util.Scanner
- 위 리소스들은 비용도 높으며, 자동으로 닫히기 어렵기 때문에 반드시 close 를 호출해야 하고, 참조가 사라지면 GC가 처리 (시간이 걸림)
- 또한, try-finally 를 사용해 close 호출함

→ 코드의 복잡성도 높아질 뿐더러, close 에서 잡히지 않는 오류가 발생 우려가 있음.

**⇒ 이 때, use 블럭을 사용 할 수 있음**

```jsx
fun CountCharactersInFile(path: String): Int {
   val reader = BufferedReader(FileReader(path))
   reader.use {
      return reader.lineSequence().sumBy { it.length }
   }
}
```

## TL;DR Closeable 또는 AutoCloseable 로 구현된 객체를 다룰 때는 use 블럭을 사용해 안전하고 간단하게 close 처리를 하자!
