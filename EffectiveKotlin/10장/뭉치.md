# 아이템 10. 단위 테스트를 만들어라

- 코드를 안전하게 만드는 궁극적인 방법: 다양한 종류의 테스트를 하는 것
- 보통은 외부에서 사용될 앱을 제대로 작동시키는 것이 목표이기 때문에 사용자 관점의 테스트만 진행함.
    
    → but, **인수 테스트(acceptance test) 는 개발자에게 필요하지 않고, 시스템의 구체적인 요소가 제대로 작동하는 지 보장해주지 않음**
    
    → 단위 테스트(unit test) 를 작성해야 함.
    
- 단위 테스트는
    1. 일반적인 유스 케이스(happy path) - 해당 요소가 사용될 거라 예상되는 일반적인 방법
    2. 일반적인 에러 케이스 혹은 잠재적인 문제
    3. 엣지 케이스와 잘못된 인수들 - Int 형의 경우 Int.MAX_VALUE 같은 매우 큰 수를 확인
- 이를 통해 구현된 요소가 동작하는 방식에 매우 빠른 피드백을 얻을 수 있어서 개발하는 동안 유용함 + 수동으로 테스트 하기 어려운 케이스도 확인할 수 있음

- 단위 테스트의 장점
    - 단위 테스트를 활용한다면 심리적인 안정감을 줄 수 있음
    - 레거시~~(고대유물..)~~를 건드릴 때 무서움을 파훼할 수 있음
    - 수동으로 테스트 하는 것보다 빠를 때가 있음
- 단위 테스트의 단점
    - 작성하는 데 시간이 걸림 → 장기적으로 봤을 때 디버깅 및 버그 찾는 데 소요되는 시간을 절약할 수 있기에 시간을 절약 시켜줄 수 있음
    - 코드를 테스트 가능하도록 수정해야 함
    - 좋은 단위 테스트를 작성하는 것은 어려움 → 단위 테스트는 API 의 명세와도 같다 볼 수 있기 때문에 잘 작성하는 것이 어려울 수 있음
