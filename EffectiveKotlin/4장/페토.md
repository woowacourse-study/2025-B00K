## inferred(추론) 타입으로 리턴하지 말라

코틀린의 타입 추론은 가장 널리 알려진 특징이다.
하지만 타입 추론을 사용할 때 몇 가지 위험한 부분들이 있다.

```
open class Animal
class Zebra: Animal()

fun main(){
  var animal = Zebra()
  animal = Animal() // Type mismatch
}
```

일반적인 경우 컴파일 에러가 발생하기 때문제 문제가 되지 않는다.
위 문제는 타입을 명시적으로 지정해 문제를 해결할 수 있다.
```
open class Animal
class Zebra: Animal()

fun main(){
  var animal: Animal = Zebra()
  animal = Animal() // Type mismatch
}
```

하지만 외부 라이브러리나 모듈같이 직접 코드를 조작할 수 없다면 어떻게 될까 ?
```
val DEFAULT_CAR: Car = Fiat126P()

interface CarFactory{
  fun produce() = DEFAULT_CAR
}
```
이 자동차 공장은 Fiat126P이라는 하나의 자동차 밖에 만들 수 없게 된다.

### 정리
- 타입을 확실하게 지정해야 하는 경우 명시적으로 타입을 지정해야 한다.
- 안전을 위해 외부 API를 만들 땐 반드시 타입을 지정하자
